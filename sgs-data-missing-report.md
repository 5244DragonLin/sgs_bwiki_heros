## 六、如何判断「网站问题」还是「爬虫代码问题」

本目录就是爬虫抓下来的结果，单看它无法直接区分"源站没给"还是"爬虫漏抓"。用以下**交叉信号**判断：

1. **经典版无技能，但同武将的皮肤页有 voice_lines 台词** → 原归因为"爬虫漏解析武将技能块"。**现确认：初版命中的 13/15 个均为爬虫 bug，已修复**；唯一剩余 SP太史慈 仍为空（网站真空缺）。
2. **经典版无技能，且皮肤页也无台词、图片也缺** → 该武将多处皆空，倾向**网站本就缺失/未上线**，或爬虫整页失败。
3. **字段在 checkpoint 里就有、但项目 `public/skin-data.json` 里没有** → 是项目 `scan-skins.js` 搬运脚本丢的（代码问题）。可运行 `npm run scan` 后对比两者差异复核。
4. **图片 ↔ metadata 精确对应**：因图片用 `-` 分隔（`武将-皮肤` 或 `皮肤-武将-变体`）、metadata 用 `*` 分隔（`皮肤*武将`），命名不统一，本审计未做精确逐一匹配，建议按第 5 节武将名单人工核对。
5. **判断流程建议**：拿 SP太史慈 → 去 biligame 三国杀武将图鉴对应页面 → 若网站上技能栏也空 → 确认是网站问题（无需改代码，或手动补 `checkpoint.json`）；其余 14 个已确认是爬虫问题，已修复。
